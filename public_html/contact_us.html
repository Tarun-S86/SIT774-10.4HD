About the Contact Form Implementation

This contact form utilizes several components to enable users to submit their inquiries efficiently:

  • Form Elements: The form consists of input fields for name, email, phone number, and message. These are created using HTML form elements with Bootstrap classes for styling.
  • Validation: JavaScript is employed for client-side validation to ensure that the user inputs are valid before the form is submitted. The validateContactForm function checks:
    • The email field contains an '@' symbol.
    • The phone number is exactly 10 digits long and numeric.
    • The message field is not empty.
  • Form Submission: Upon successful validation, the form data is sent to the server at the /contact endpoint using a POST request. This allows the server to process the data.
  • Database Integration: The submitted form data is then stored in a MySQL database. The information captured includes:
    • Name: The user's name.
    • Email: The user's email address for follow-up.
    • Phone Number: Optional for direct contact.
    • Message: The content of the user's inquiry or feedback.
    This allows for efficient management of user inquiries and facilitates follow-up communications.
